## 3. Problem Analysis

### 3.1 Pathological Churn (18x Daily Turnover)

547 fills today is extreme for a $100k account. Root causes:

| Factor | Contribution |
|--------|-------------|
| 4 overlapping equity strategies on same 10 symbols | All fire independently on 1-min bars. `exclusive_per_symbol=false` means avwap_v1, avwap_v2, ai_scalping, and ORB can ALL generate signals for TSLA simultaneously |
| LLM revaluator tightening everything | Every 5 min, stops get 0.7x tighter → premature stop-outs → 60s cooldown → immediate re-entry |
| 60-second exit cooldown is trivially short | For a 1-min bar system, this allows re-entry on the very next bar |
| Partial fills inflating count | Limit orders fill in slices (e.g., TSLA: 1 share, 1 share, 5 shares, 0.84 shares = 4 fills for one $10k entry) |
| max_trades_per_day is per-strategy | avwap_v1 (3/day) + avwap_v2 (3/day) + ai_scalping (10/day) + ORB (1/day) = 17 trades/day/symbol theoretical max |

**Industry benchmarks:**
- Standard daily turnover for intraday scalping: 1–3x notional
- Break-even alpha at 18x turnover with 5 bps costs: >0.5% per trade (unsustainable for 1-min scalping)
- Reference: *Optimal trend-following with transaction costs* (Zakamulin & Giner, 2024)

### 3.2 Position Sizing Math Is Broken

```
Current:  $100 risk (10 bps) / ($price × 0.0025 stop) = $40,000 position → clamped to $10,000
          Effective risk = $10,000 × 0.0025 = $25 per trade (2.5 bps)
          This is LESS than the ~$50 execution cost (5 bps round-trip)

Fixed:    ATR-based stop (e.g., 2× ATR for SPY ≈ 50-80 bps)
          $1,000 risk (100 bps) / ($price × 0.006 stop) = ~$10,000 position (fits naturally)
          Effective risk = $1,000 per trade — meaningful edge capture
```

The risk sizer calculates a $40k position then clamps to $10k every single time. This means the max_position_bps cap is the de facto position sizer, and the risk-per-trade math is irrelevant.

**Industry benchmarks:**
- Kelly Fraction: Professional consensus is 0.1–0.25 (Fractional Kelly)
- Stop distance: Standard is 1.5–3.0× ATR(14), not static bps
- Reference: QuantConnect Lean ATR implementation, QuantStrategy ATR sizing guide

7 of 9 positions marked DEGRADING. This creates a death spiral:
1. LLM marks position DEGRADING (which it does for almost everything at 0.60+ confidence)
2. Stops tighten by 30% (0.7x multiplier)
3. Tight stops trigger exit on normal 1-min noise
4. 60s cooldown expires → strategy re-enters on next signal
5. Revaluation gate blocks for 10 min, then expires → cycle repeats
6. Each round-trip leaks 3-5 bps in spread/slippage

**Industry benchmarks:**
- Risk assessment interval should match Expected Trade Life — 15-30 min for intraday, not 5 min
- Confidence threshold for action: >0.85 (current 0.60 threshold is too permissive)
- Tightening cap: max 10% of remaining distance per cycle
- References: QuantAgent (2026, OpenReview), Zakamulin adaptive stops research

Root cause: Dropped WebSocket partial fills or order execution callbacks. The negative AVAX qty means the DB recorded more SELL fills than BUY fills — some BUY fills were lost.

Impact: Position gate and revaluation gate use DB state. Circuit breaker checks realized P&L from the ledger — if fills are missing, daily P&L calculation is wrong.

**Industry best practice:**
- Broker is Source of Truth (SoT), local DB is a cache
- 3-layer reconciliation: real-time WS + 5-min REST audit + on-startup full replay
- Alpaca SSE provides replayable event streams with monotonic IDs
- Alpaca FIX API provides sequence numbers for gap recovery
- References: Alpaca SSE docs, Alpaca FIX announcement

**Industry benchmarks:**

### 3.6 Strategy Overlap

With `exclusive_per_symbol=false`, all 4 equity strategies fire independently:

```
TSLA signal timeline (hypothetical 1-min):
  09:31 - ORB detects breakout    → BUY signal (priority 100)
  09:32 - avwap_v1 breakout       → BUY signal (priority 80)
  09:32 - avwap_v2 breakout       → BUY signal (priority 80)
  09:33 - ai_scalping RSI<30      → BUY signal (priority 60)
```

Position gate prevents doubling, BUT once the position exits, all 4 race to re-enter. The `max_trades_per_day` counter is per strategy instance, so combined daily capacity is 3+3+10+1 = 17 round-trips per symbol.

**Industry best practice:** Use a Global Portfolio Manager / Signal Deconfliction Layer that aggregates and nets signals before execution. Reference: QuantConnect PortfolioConstructionModel.

## 4. Systemic Risks for Live Deployment

1. **Correlated drawdown wipeout** — $90k in high-beta tech + crypto with only a $5k daily loss breaker. A flash crash (-7% SPY) would blow through the circuit breaker before it triggers.

2. **State desynchronization** — DB and broker are already diverged after 5 days of paper trading. In live trading, phantom positions and missing fills would cause order rejections, incorrect sizing, and potential margin violations.

3. **Adverse selection on limit entries** — Limit orders 5-8 bps above ref price during momentum breakouts only fill on pullbacks. This systematically enters at worse prices than expected, and tight 25 bps stops don't survive the pullback that filled you.
